# Параметры
## Правила для значений по умолчанию:
- Можно указывать для параметров методов, конструкторов методов, параметрических свойств, параметров, являющихся частью определения делегатов;
- Параметры со значениями по умолчанию должны следовать после параметров без значений по умолчанию, но до params;
- params не может иметь значений по умолчанию;
- во время компиляции значения по умолчанию должны оставаться неизменными, 
поэтому значения по умолчанию можно определить для примитивных типов, перечислимых типов и ссылочных типов, допускающих значение null.
- задать значение по умолчанию для значимых типов можно как при помощи ключевого слова new, так и при помощи ключевого слова default, при этом генерируется IL-код, в котором все поля значимого типа заполняются нулями.
- запрещено переименовывать параметрические переменные, так как это влечет за собой необходимость редактирования вызывающего кода, который передает аргументы по имени параметра;
- для ref и out значения по умолчанию не создаются;
- при изменении порядка необходимо, чтобы именованные аргументы шли после неименованных;
## ref и out - передача ссылки на область памяти
ref - параметр должен быть инцииализирован до вызова метода. Метод может как читать, так и записывать значение параметра.</br> 
out - вызывающий код может не инициализировать его, пока не вызван сам метод. 
Вызванные метод не может прочитать значение параметра и должен присвоить ему значение перед возвращением управления.</br>
## params
Ключевое слово params позволяет указывать в качестве аргумента неопределенное кол-во параметров.
Метод, содержащий params, вызывается в том случае, когда остальные методы класса не подходят по сигнатуре.
**В качестве аргументов лучше использовать наиболее обобщенные типы (наиболее удачно подходят интерфейсы), 
а возвращать наиболее сильный объект, чтобы его функциональность была "максимальной".**

## Анонимные типы
var anonymusTypeName = new { Prop1 = Val, Prop2 = Val2 } - синтаксис объявления анонимного типа в языке C#. 
У анонимных типов нельзя изменить значения свойств, нельзя передавать в качестве параметра метода и объявлять в качестве возвращаемого значения.

## Индексаторы
При помощи индексаторов можно перегружать оператор [] у объектов.</br>
public ReturnType this[] { get{} set{} } - синтаксис определения индексатора в C#.
Индексатор может принимать и возвращать любые значения. 
Один тип может иметь несколько перегружаемых индексаторов, если они различаются сигнатурой.
Компилятор присваивает имена set_Item() и get_Item() для методов индексаторов.
При помощи атрибутов можно задавать имена get и set методов, однако C# не позволяет определить методы с разными названиями, но с одинаковой сигнатурой.
## Производительность индексаторов и свойств
В релизной версии программы компилятор вставлят код индексаторов в код, где происходит вызов индексатора (inline), в отличии от дебажной версии, 
поэтому **скорость обработки индексаторов в релизной версии выше.**
Доступ же к свойствам не зависит от типа компиляции.
